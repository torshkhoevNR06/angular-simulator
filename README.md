# 🚀 IT Simulator

> **Учебный проект** для повышения навыков веб-разработки и глубокого изучения современных технологий.

---

## 📝 О проекте

**IT Simulator** — это интерактивное приложение, разработанное как учебный проект для:

- ✨ Повышения практических навыков в **Angular** и современной архитектуре приложений
- 📚 Изучения **TypeScript** и объектно-ориентированного программирования
- 🎨 Освоения стилизации с помощью **SCSS** и создания адаптивного дизайна
- 🔌 Понимания работы **сервисов** и компонентной архитектуры
- 🌐 Взаимодействия с API и управления состоянием приложения

---

## 🛠️ Используемые технологии

- **Angular** — современный фреймворк для создания SPA
- **TypeScript** — типизированный язык программирования
- **SCSS** — препроцессор CSS для удобной стилизации
- **Routing** — клиентская маршрутизация
- **RxJS** — реактивное программирование
- **Local Storage API** — хранение данных на клиенте

---

## 🔧 Применённые возможности Angular

В проекте используются ключевые концепции и инструменты Angular для построения масштабируемого приложения:

### **Сервисы** 📦
- `UserService` — управление данными пользователей с использованием BehaviorSubject
- `LoaderService` — контроль состояния загрузки и управление DOM элементами
- `MessageService` — система уведомлений с автоматическим скрыванием сообщений
- `UserApiService` — взаимодействие с backend API
- `LocalStorageService` — работа с хранилищем браузера

### **Декораторы** 🏷️
- `@Injectable()` — создание сервисов с автоматическим внедрением зависимостей
- `@Component()` — определение компонентов с шаблонами и стилями
- `providedIn: 'root'` — регистрация сервисов на уровне приложения

### **Dependency Injection (DI)** 💉
- Функция `inject()` — современный способ внедрения зависимостей в компонентах и сервисах
- Типизированные ссылки на сервисы для лучшей типобезопасности

### **Реактивное программирование (RxJS)** ⚡
- `BehaviorSubject` — управление состоянием с последним значением
- `Observable` — реактивные потоки данных
- Операторы: `pipe()`, `catchError()`, `finalize()` — обработка асинхронных операций
- Подписки в шаблонах через `async`-трубку

### **Маршрутизация** 🗺️
- `Routes` — конфигурация маршрутов приложения
- `<router-outlet>` — динамическая загрузка компонентов страниц
- Wildcard маршруты `**` — обработка несуществующих страниц

### **Компонентная архитектура** 🧩
- Smart-компоненты (страницы): `HomePageComponent`, `UsersPageComponent`
- Dumb-компоненты (переиспользуемые): `HeaderComponent`, `FooterComponent`, `LoaderComponent`, `MessageComponent`
- Принцип разделения ответственности (SoC)

### **Типизация с TypeScript** 📝
- Интерфейсы для всех сущностей приложения (IUser, IMessage, INavigation и т.д.)
- Перечисления (Enums): `MessageType`, `Color` для безопасных значений
- Типизированные методы сервисов
